# UI Patterns

本文件记录通用 UI / HTML / 可视化任务规则。具体设计系统由项目 adapter 或应用真源定义。

## 用户可见任务

如果用户要求筛选、搜索、排序、操作入口、看板、页面变化或可点击 demo，默认理解为用户可见能力。

实现前必须写清：

- 用户第一眼看到什么
- 用户能操作什么
- URL 或产物路径
- 刷新或重新生成方式
- 空态和错误态
- 本轮明确不做什么

## QA 要求

UI / HTML / 可视化任务不能只靠 `rg`、JSON、HTML 字符串或截图外观标记完成。

需要尽量使用 Browser、Chrome、Playwright 或等价方式实际打开并操作：

- 控件可见
- 控件可操作
- 状态变化正确
- 空态正确
- 响应式行为可接受
- 刷新 / 重新生成说明清晰

如果浏览器验证被权限、环境或文件协议阻断，改用本地 HTTP server。仍无法验证时，QA 只能标 `blocked` 或 `partial`。
