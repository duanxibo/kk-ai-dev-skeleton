# State Management

本文件记录通用状态管理原则。具体状态库、服务层和数据流由 adapter 和项目真源定义。

## 默认原则

- 持久状态属于后端、服务层或项目指定的数据真源。
- 派生业务状态优先在服务层或明确的 shared contract 中定义。
- 前端只持有瞬时 UI 状态、输入状态、加载状态和用户临时选择。
- URL 状态适合表达可分享、可刷新、可回放的筛选或定位。
- 本地缓存必须有失效策略和刷新说明。

## QA 要求

状态相关任务需要验证：

- 初始状态
- 加载状态
- 空态
- 错误态
- 刷新后的状态恢复
- scope 切换后的清理
- 多次操作后的稳定性

如果用户反馈“刷新后没变化”，先确认本轮是否真的改变了用户可见页面，再检查产物是否重新生成、dev server 是否重启、浏览器是否打开旧文件。
