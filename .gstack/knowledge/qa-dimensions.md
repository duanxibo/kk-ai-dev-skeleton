# QA 维度

QA evidence 应证明这次任务真正重要的行为。

## 通用检查

- 任务始终在 boundary 内。
- forbidden paths 没有被触碰。
- required gates 已完成、明确不需要、blocked，或带原因 deferred。
- 真源文档已更新，或明确说明不需要。
- 验证命令和结果已记录。
- 残余风险已列出。

## UI / HTML / 可视化任务

不要只靠 `rg`、JSON、HTML 字符串或截图外观就把 UI 任务标记完成。

需要记录：

- URL 或产物路径
- 刷新或重新生成步骤
- 结构存在
- 控件可见
- 控件可操作
- 状态变化正确
- 空态正确
- 错误可理解
- 如相关，响应式行为可接受
- Browser、Chrome、Playwright 或等价交互证据

如果直接浏览器验证被阻断，记录尝试过什么，并把结果标记为 `blocked` 或 `partial`。

## API / 数据任务

需要记录：

- 输入输出契约
- scope
- 正常情况
- 空数据情况
- 非法输入
- 权限或授权行为
- 数据源和新鲜度假设
- 除非 adapter 明确允许，否则前端不得直接依赖数据库

## 纯文档任务

需要记录：

- 改了哪些文档
- 用户或 agent 行为有什么变化
- 未来用户如何验证这次改动
