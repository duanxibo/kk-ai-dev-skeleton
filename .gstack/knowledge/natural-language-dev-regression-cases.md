# 自然语言开发回归用例

这个文件定义不能退化的用户旅程。

运行：

```bash
python3 .gstack/scripts/natural_language_dev_smoke.py
python3 .gstack/scripts/natural_language_dev_smoke.py --format user
```

## 必须覆盖的用户旅程

- 打开开发首页。
- 询问已经做了什么。
- 询问现在能不能开始实现。
- 询问当前需要确认什么。
- 只做模糊确认。
- 暂停当前任务。
- 请求撤销或回滚。
- 索要需求模板。
- 询问填好的需求是否完整。
- 询问如何描述已有页面改动。
- 索要首次使用引导。
- 不想选择技术路线时，让 Codex 给推荐方案。
- 调整任务范围。
- 要求继续推进。
- 查看任务清单。
- 询问某个具体需求如何验收。
- 反馈看不到页面变化。
- 反馈 CI 失败。
- 索要交付总结。
- 要求团队状态同步，但不引入数据库。

## 通过标准

面向用户的输出应隐藏内部路径、raw JSON、boundary 机制和 gate 名称，除非用户明确要求工程细节。

helper 层可以使用内部 evidence，但用户看到的应该是清晰下一步。
