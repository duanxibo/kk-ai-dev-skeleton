# Blueprint

`blueprint/` 是前置蓝图层。

它用于记录项目在进入正式工程实现前的系统级构想、核心对象、设计原则、阶段演进和关键取舍。

## 放什么

- 系统为什么存在。
- 主要用户和核心场景。
- 核心对象、边界和关系。
- 初版设计原则。
- 阶段演进路径。
- 尚未进入正式 stack spec 的开放问题。

## 不放什么

- 具体实现代码。
- API / DTO / 数据表的最终契约。
- 测试 fixture 和业务脚本。
- 已经冻结并进入开发的模块规格。

进入正式实现前，采纳项应同步到 `stack/<project>/specs/`、adapter 或 `.gstack/` 对应 evidence。
